## 1. 벡터

### 1.1 벡터 생성

벡터는 R에서 가장 기본적인 데이터 구조이다. 벡터는 **동일한 유형**의 원소로 구성된 1차원 순서형 집합이다.

```r
# c() 함수를 사용하여 벡터 생성
ex_vector1 <- c(-1, 0, 1)          # 수치형 벡터
ex_vector2 <- c("Hello", "Hi~!")   # 문자형 벡터
a <- c(1, 2, 9)                    # 수치형 벡터

# 수열 생성
1:10                               # 1부터 10까지의 정수 수열
seq(1, 9, by = 2)                  # 수열: 1, 3, 5, 7, 9
```

### 1.2 벡터 속성

```r
mode(ex_vector1)    # "numeric"   - 저장 모드
mode(ex_vector2)    # "character" - 저장 모드

str(ex_vector1)     # num [1:3] -1 0 1
str(ex_vector2)     # chr [1:2] "Hello" "Hi~!"

length(ex_vector1)  # 3
length(ex_vector2)  # 2
```

### 1.3 벡터 연산

```r
a <- c(1, 2, 9)
mean(a)             # 4 (산술 평균)

# 벡터화 연산
x <- c(1, 2, 3)
x + 2               # 3, 4, 5 (원소별 덧셈)
x * 3               # 3, 6, 9 (원소별 곱셈)
```

> **핵심 요점:** R의 벡터 연산은 **벡터화(vectorized)** 되어 있어, 명시적인 반복문 없이 각 원소에 자동으로 적용된다.

## 2. 행렬

### 2.1 행렬 생성

행렬은 모든 원소가 동일한 유형이어야 하는 2차원 데이터 구조이다.

```r
x <- c(1, 2, 3, 4, 5, 6)

# 기본값: 열 기준으로 채움 (열 우선 순서)
matrix(x, nrow = 2, ncol = 3)
#      [,1] [,2] [,3]
# [1,]    1    3    5
# [2,]    2    4    6

# 행 기준으로 채우기
matrix(x, nrow = 2, ncol = 3, byrow = TRUE)
#      [,1] [,2] [,3]
# [1,]    1    2    3
# [2,]    4    5    6
```

### 2.2 행렬 인덱싱

```r
a <- matrix(c(1,2,3,4,5,6), nrow = 2, ncol = 3)
a[1, 2]     # 1행 2열의 원소 -> 3
a[1, ]      # 첫 번째 행 전체
a[, 2]      # 두 번째 열 전체
```

### 2.3 행렬 확인 및 변환

```r
a1 <- 1:10
is.matrix(a1)              # FALSE (벡터임)

a2 <- as.matrix(1:10)
is.matrix(a2)              # TRUE

is.matrix(as.matrix(1:10)) # TRUE
```

> **핵심 요점:** R은 기본적으로 행렬을 열 단위로 채운다 (열 우선 순서). 행 단위로 채우려면 `byrow = TRUE`를 사용한다.

## 3. 배열

### 3.1 배열 생성

배열은 행렬을 2차원 이상으로 확장한 구조이다.

```r
y <- c(1, 2, 3, 4, 5, 6)

# 3차원 배열 생성: 2행 x 2열 x 3층
array(y, dim = c(2, 2, 3))
```

`dim` 인수는 차원을 지정한다:
- 첫 번째 값: 행의 수
- 두 번째 값: 열의 수
- 세 번째 값: 층(깊이)의 수

### 3.2 배열 vs. 행렬

```r
# 배열은 다차원이며, 행렬은 2차원 배열이다
is.array(matrix(1:6, 2, 3))  # TRUE (행렬은 2차원 배열이다)
is.matrix(array(1:6, c(2,3))) # TRUE (2차원 배열은 행렬이다)
```

## 4. 리스트

### 4.1 리스트 생성

리스트는 R에서 가장 유연한 데이터 구조이다. 벡터나 행렬과 달리, 리스트는 **서로 다른 유형과 크기**의 원소를 포함할 수 있다.

```r
list1 <- list(c(1, 2, 3), "Hello")
list1
# [[1]]
# [1] 1 2 3
#
# [[2]]
# [1] "Hello"

str(list1)
# List of 2
#  $ : num [1:3] 1 2 3
#  $ : chr "Hello"
```

### 4.2 명명된 리스트

```r
pts <- list(x = cars[,1], y = cars[,2])
is.list(pts)    # TRUE
is.matrix(pts)  # FALSE
is.array(pts)   # FALSE
```

### 4.3 혼합 유형 저장

```r
# 리스트만이 서로 다른 유형과 길이의 데이터를 묶을 수 있다
x <- list("a", "b", 1:10)
length(x)  # 3 (리스트에 3개의 원소)
```

> **핵심 요점:** 리스트는 R에서 서로 다른 유형과 길이의 원소를 담을 수 있는 **유일한** 데이터 구조이다. 따라서 복잡하고 이질적인 데이터를 저장하는 데 필수적이다.

### 4.4 중첩 리스트

```r
x1 <- data.frame(var1 = c(1, 2, 3), var2 = c("a", "b", "c"))
x2 <- matrix(1:12, ncol = 2)
x3 <- array(1:20, dim = c(2, 5, 2))

x4 <- list(f1 = x1, f2 = x1, f3 = x2, f4 = x3)
class(x4)   # "list"
x4$f1       # 이름으로 첫 번째 원소에 접근
```

## 5. 데이터 프레임

### 5.1 데이터 프레임 생성

데이터 프레임은 R에서 표 형태 데이터를 다루는 주요 구조이다. 본질적으로 동일한 길이의 벡터들로 구성된 리스트이며, 각 열은 서로 다른 유형을 가질 수 있다.

```r
ID <- c(1, 2, 3, 4, 5)
GENDER <- c("F", "M", "F", "M", "M")
AGE <- c(50, 40, 28, 50, 27)
AREA <- c("Seoul", "Gyeonggi", "Jeju", "Seoul", "Seoul")

dataframe_ex <- data.frame(ID, GENDER, AGE, AREA)
```

### 5.2 데이터 프레임 확인

```r
dataframe_ex            # 데이터 프레임 출력
str(dataframe_ex)       # 구조: 유형, 차원
View(dataframe_ex)      # 스프레드시트 뷰어에서 열기
nrow(dataframe_ex)      # 행의 수
ncol(dataframe_ex)      # 열의 수
```

### 5.3 열 접근

```r
# $ 연산자 사용
dataframe_ex$AGE        # AGE 열을 벡터로 반환
mean(dataframe_ex$AGE)  # AGE 열의 평균 계산

# 대괄호 표기법 사용
dataframe_ex[, 3]       # 세 번째 열 (AGE)
dataframe_ex[1, ]       # 첫 번째 행
dataframe_ex[2, 3]      # 2행 3열의 원소
```

### 5.4 대안: data.frame() 인라인 생성

```r
x1 <- data.frame(
  var1 = c(1, 2, 3),
  var2 = c("a", "b", "c")
)
class(x1)  # "data.frame"
```

> **핵심 요점:** 데이터 프레임은 R에서 데이터 분석의 핵심 구조이다. 각 열은 변수, 각 행은 관측치로, 표준 정돈 데이터(tidy data) 형식과 일치한다.

## 6. 팩터

### 6.1 팩터란?

팩터는 **수준(levels)**이라 불리는 고정된 값의 집합을 가진 범주형(질적) 데이터를 나타낸다.

```r
blood.type <- factor(
  c("A", "B", "AB", "O", "O"),
  levels = c("A", "B", "AB", "O")
)
class(blood.type)  # "factor"
str(blood.type)    # Factor w/ 4 levels "A","B","AB","O": 1 2 3 4 4
table(blood.type)  # 빈도표
# A  B AB  O
# 1  1  1  2
```

### 6.2 팩터 vs. 문자형 벡터

```r
blood2 <- c("A", "B", "AB", "O", "O")
class(blood2)   # "character"
table(blood2)   # 빈도표가 생성되지만 고정된 수준이 없음
```

### 6.3 팩터의 산술 연산 동작

```r
var1 <- c(1, 2, 3, 1, 2)
var2 <- factor(c(1, 2, 3, 1, 2))

var1 + 2    # 작동함: 3, 4, 5, 3, 4
var2 + 2    # 경고! 팩터는 수치형이 아님

mean(var1)  # 1.8
mean(var2)  # NA (경고 발생)

# 팩터를 수치형으로 변환
var2_num <- as.numeric(var2)
mean(var2_num)   # 내부 코드(1, 2, 3, 1, 2)의 평균을 반환
```

### 6.4 팩터에서의 대소문자 구분

```r
blood <- factor(
  c("A", "a", "B", "AB", "o", "O"),
  levels = c("A", "B", "AB", "O")
)
table(blood)
# A  B AB  O
# 1  1  1  1
# 참고: "a"와 "o"는 어떤 수준과도 일치하지 않으므로 제외됨
```

> **핵심 요점:** 팩터는 내부적으로 정수와 관련된 수준 레이블로 데이터를 저장한다. 산술 연산에 직접 사용할 수 없으며, 먼저 `as.numeric()`으로 수치형으로 변환해야 한다.

## 7. 유형 확인 및 변환

### 7.1 유형 확인

```r
is.vector(c(1, 2, 3))     # TRUE
is.matrix(matrix(1:4, 2))  # TRUE
is.list(list(1, "a"))      # TRUE
is.data.frame(mtcars)      # TRUE
is.factor(factor("A"))     # TRUE

class(c(1, 2, 3))          # "numeric"
class(TRUE)                # "logical"
class(factor("A"))         # "factor"
```

### 7.2 유형 변환 함수

| 함수 | 변환 대상 |
|------|----------|
| `as.numeric()` | 수치형 |
| `as.character()` | 문자형 |
| `as.logical()` | 논리형 |
| `as.factor()` | 팩터 |
| `as.matrix()` | 행렬 |
| `as.data.frame()` | 데이터 프레임 |

### 7.3 내장 데이터셋

R에는 연습용으로 많은 내장 데이터셋이 포함되어 있다:

```r
data()         # 사용 가능한 모든 데이터셋 목록
cars           # 자동차의 속도와 제동 거리
mtcars         # 자동차 데이터 (32대, 11개 변수)
airquality     # 뉴욕의 대기질 측정 데이터
economics      # 미국 경제 시계열 데이터
```

## 요약

| 데이터 구조 | 차원 | 허용되는 유형 | 주요 함수 |
|------------|------|--------------|----------|
| 벡터 | 1차원 | 동일 유형만 | `c()` |
| 행렬 | 2차원 | 동일 유형만 | `matrix()` |
| 배열 | n차원 | 동일 유형만 | `array()` |
| 리스트 | 1차원 (중첩 가능) | 다른 유형 가능 | `list()` |
| 데이터 프레임 | 2차원 | 열별로 다른 유형 가능 | `data.frame()` |
| 팩터 | 1차원 | 범주형 수준 | `factor()` |

| 개념 | 핵심 요점 |
|------|-----------|
| 벡터 | 기본 1차원 구조; 모든 원소가 동일 유형 |
| 행렬 | 2차원; 기본 채움 방식은 열 우선 (`byrow = TRUE`로 행 우선 변경) |
| 리스트 | 혼합 유형과 길이를 담을 수 있는 유일한 구조 |
| 데이터 프레임 | 표 형태 데이터 (행 = 관측치, 열 = 변수) |
| 팩터 | 고정된 수준을 가진 범주형 데이터; 산술 연산에 직접 사용 불가 |
| `$` 연산자 | 데이터 프레임과 리스트에서 명명된 원소에 접근 |
